  • From day01, contents vs fields?
    Fields are like properties to the class.
    For example, address contents have state, country, city,
    postbox, street address as components, and they are
    fields in drupal.
    Terms
  • View Mode = How your page is displayed.
  • field format = .
  • CMS = Content Management System
  • Module = A set of PHP, Javascript, and CSS extending site features and add functionality.
  • Themes = A set of files defining the visual look and feel of your website.
  • Distribution = provision of site features and functions for a specific type of site as single downlad.
  • Contents Entity(entity) = items for contents type.
  • Fields = sub-type and properties for entities.
  • URL = Uniform Resource Locator aka address of the web site.
  • Path = unique part of URL for specific function and contents.
  • Alias = more understandable name for the paths.
    Tips
  • You can reuse the field from other content type.
  • Simply go to configuration -> basic site settings and set up default front page.
  • You can use quick editor for editing the contents.
  • If change is not applied immediately, clear the cache.
  • In order to use admin tool bar,
    • lando drush uli //if user account is not set up
    • copy the uri.
    • lando info -> get site's url
    • paste it into the default
  • When making a boiler plate website using Lando ,lando info and get the information you need.
  • Database can be either mySQL(5.5.3 or higher) with InnoDB compatibility
  • Apache is recommended for web servers.
  • You need PHP 7.2 or higher.
  • You can easily change contents on drupal by installing and uninstalling modules
  • Don’t forget to remove data or configuration for function or features to be removed.
  • remove the modules if not used.
  • Make mobile site layout first since people uses phone first often.
  • Making modular data helps maintain or update the page easily.
